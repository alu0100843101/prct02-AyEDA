Día: 2.3.2016

La práctica se realizará en notación postfija.

3+2 --> 3 2 +
(3+2)*5 --> 3 2 + 5 *

La estructura que va a ser usada es la Pila. 
Se almacenan operandos, operadores y el resultado.

Los operandos pueden ser: números racionales, enteros o complejos. 
Hay que crear tres clases: Racional, Entero y Complejo.

Hay que usar templates para establecer el tipo de dato para que no coja siempre 
el mismo tipo, sino que pueda seleccionarlo.

Funciones Friends: Sirven para usar funciones para de otras clases sin tener que 
implementarlas en la clase que estamos implementando.

Ejemplo: 

Class B
{
    ...
    bool recorrer(V, id);
    ...
}

Class A
{
    ...
    friend bool recorrer(); 
    //Podrá usar la funcion de la clase B, no implementada en A.
    ...
}

La rúbica de calificaciones será similar a la rúbica de la primera práctica. 

Insiste mucho en "modular" las clases. 

Puede ser que exista una clase calculadora con un stack
o una función calculadora en stack.

Si se añade polimorfismo o herencia, son opciones que se valoran. 

IDES recomendados: Netbeans o Eclipse. Pero se puede usar cualquier IDE.

Realizar tests unitarios: Con uno sobre cualquier cosa basta. 

Repositorio github o bitbucket.
